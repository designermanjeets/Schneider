Using Cascading Parameters

Xanbus-enabled devices are typically configured one at a time. However, because devices often have common settings, cascading parameters can simplify and speed up system configuration. When a parameter is “cascaded,” a newly configured device setting on one device is applied to all devices of the same device type on a common AC or DC connection.

Table 3-1 shows which device settings are able to be cascaded. Once cascading parameters are enabled, any change to a device setting in listed in Table 3-1 will be automatically applied to all devices of the same type.

For example, if cascading parameters are enabled and a Battery Size (Ah) parameter is changed on the XW inverter setup menu, the same battery size setting is applied to other XW inverters.

To enable or disable cascading parameters:

Put the device on standby mode. See “To set a device in Standby mode:” on page 3–38.

Click System Devices to access the individual devices menu.

Click the device you want to configure.

Click from the menu tree below the highlighted device.

You will see a list of Device Settings for the device.

:

Device settings for each Xanbus device vary and may not contain the Cascading Settings option. Be sure to consult the Owner’s Guide of the device before trying to change the settings via the Conext Gateway. Also, when configuring the network-managed power system with cascading parameters enabled, you should verify that the configurations have been cascaded correctly to other devices. Communication errors may cause parameters to revert to their original values or generate a warning message.

Click to expand the device setting.

Click the Enable Cascaded Settings (All Devices) check box to enable the setting. If the box is already checked then cascading is already set up and no further action is required.

Click to save the change.
