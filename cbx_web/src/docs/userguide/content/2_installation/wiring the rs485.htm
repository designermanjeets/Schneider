Wiring the RS 485 Modbus Connector for Data Communication to the Conext Gateway

The RS 485 Modbus connector provides three terminals to wire communication cables to the Conext Gateway. The five-terminal RS 485 connector uses the first three terminals for a data cable. The cable has a size of 16–24 AWG with 1.5 mm2–0.25 mm2 wires. The cable can be shielded or non-shielded.

To wire the RS 485 Modbus connector for data communication:

Select a two-wire, twisted pair, shielded cable not longer than 164 feet (50 m). Refer to the local electrical code and application to select the insulation and temperature class of the cable to be used.

Strip 3/8 in.(10 mm) from the end of the wires to be connected and attach ferrules to the two signal wires (red and black in the following diagram).

Insert the ferrules and the shield wire into the connector terminals as shown in the following diagram.

Secure the wires by tightening the screw on the terminal.

The middle terminal is not connected internally but is provided for the shield connection of the cable.

Modbus versus Xanbus

The RS 485 Modbus connection and Xanbus cable connection provide data communication from the network and devices to the Conext Gateway. Communication with Modbus devices is handled through the RS 485 or 10/100 Ethernet connection on the Conext Gateway while communication with Xanbus components occurs through the Xanbus ports of Xanbus-enabled devices.

Connecting the Conext Gateway with other Modbus Devices

In a Modbus implementation, the Conext Gateway can act as a master or slave to an RS 485 master device. The RS 485 connection to the Conext Gateway allows communication between the Xanbus network and the third- party device. This enables Conext devices to link to third-party software and building management systems.

If a Modbus device, including the Conext Gateway, is installed as the last device in a daisy chain, a 120 ohm terminator must be used because they do not have an internal terminator for the RS 485 network.
